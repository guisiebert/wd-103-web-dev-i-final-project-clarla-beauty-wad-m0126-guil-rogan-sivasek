# Final Project

**Objective:**

The objective is to create the website using the Figma mockup. Access to the mockup will be shared via email invite, or you may use the link below:

[Click here](https://www.figma.com/design/HSnOpO5p4b4LWaUs08Pbm6/BC-Beauty---WMAD-UI-UXDesign--1st-Version----WINNER?node-id=4115-1345&t=h9kbkdvfrTOW1gBH-1)

The website consists of the following pages: 
* Home  
* My Work  
* Services  
* Classes  
* BeautyTips  
* About  
* Contact  

This exercise will help sharpen your HTML, CSS, and JS skills, and also improve your ability to work effectively as a team.

---

## Evaluation Criteria

- **Code (HTML, CSS, JS):** 40%  
  Code quality, structure, responsiveness, and implementation

- **Design Accuracy:** 30%  
  How closely your final website matches the Figma mockup

- **Presentation:** 30%  
  Clarity, communication, and ability to explain your work

---

_Note: There are 8 pages, but the main ones are HOME, CLASSES, and CONTACT pages. Ensure these are completed before moving on to other pages or sections._

---

## Project Details

- Convert the Figma design into a fully functional website  
- Discuss and clearly divide work among team members, and stick to your responsibilities  
- Use Git for version control, commit frequently  
- Set branch protection rules for main, no direct pushes to main  
- Use HTML, CSS (SASS), and JavaScript  
- Any forms included in the website should accept user input and provide feedback on submission  
- On submission, form values should be handled through JavaScript and at minimum be shown through `console.log()` and/or `alert()`  

---

## Technical Restrictions

- Do not use frameworks such as React, Astro, or similar  
- Do not use CSS libraries or frameworks such as Tailwind, Bootstrap, or others  
- You may use jQuery  
- You may use third-party libraries (e.g., Swiper) if necessary to achieve design accuracy  
- It is recommended to consult with your instructor before using any third-party library  

---

## Code Integrity

- All work must reflect your team’s own understanding and effort  
- You may be asked to explain any part of your code during the presentation  
- Be prepared to justify your implementation decisions and demonstrate how your code works  

---

## Deployment Requirement

- Your team must deploy the final project on a hosting provider such as Netlify or Vercel  
- You must present your project using the live deployed link during the presentation  

---

## Presentation Details

- **Duration:** ~15 minutes per team (excluding Q&A)

- **You should cover:**
  - Project overview  
  - How work was divided among team members  
  - Code structure and key implementation decisions  
  - Live demo of the deployed project  
  - Biggest struggles or challenges faced
